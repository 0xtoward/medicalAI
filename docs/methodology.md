# 研究方法（校正）

## 1. 研究设计与研究队列

本研究为单中心回顾性队列研究。研究对象为 2021 年 3 月至 2024 年 6 月在天津医科大学总医院接受放射性碘治疗（radioactive iodine, RAI）的 Graves 病甲状腺功能亢进症患者。所有患者均按照常规临床流程进行随访，随访时间节点包括基线（0 月）以及治疗后 1、3、6、12、18 和 24 个月。纳入标准包括：符合相关诊疗规范；首次接受 RAI 治疗；治疗前停用抗甲状腺药物达到预设时间要求；具有可用于纵向分析的随访资料。排除标准包括：合并可疑恶性甲状腺结节或甲状腺恶性肿瘤；既往接受过 RAI 治疗；既往行甲状腺切除术或颈部放射治疗；无法用于时序建模的资料缺失情形；治疗后再次使用抗甲状腺药物。所有数据均来源于医院电子病历系统及随访数据库。本研究经天津医科大学总医院伦理委员会批准，因属回顾性研究而豁免患者知情同意。

## 2. 状态定义

在每个随访时间点，临床医生根据甲状腺激素水平和临床评估将患者甲状腺状态划分为三种互斥类别：甲亢（Hyper）、正常（Normal）和甲减（Hypo）。状态判定基于 `FT3`、`FT4`、`TSH` 等指标及临床综合判断。上述三状态构成纵向疾病轨迹分析的基础。对于存在随访缺口的情形，本研究不使用未来信息进行人工回填；相关预测任务仅在能够由当前时点可见信息形成有效输入和结局判定的时间点或时间区间上进行建模。对于输入协变量本身的缺失，则在后续时间安全预处理框架下进行处理。

## 3. 临床预测目标

本研究设置两类主要预测任务。首先，进行固定时间点预测（fixed landmark prediction），用于评估早期治疗反应，即在治疗后 3 个月和 6 个月时识别患者当前是否仍处于甲亢状态（Hyper vs Non-Hyper）。在此基础上，进一步扩展为 `Hyper / Normal / Hypo` 三分类状态分流任务。其次，采用滚动时间点预测（rolling landmark prediction）构建动态复发预测模型。对于在时间点 `t_k` 处于 Normal 状态的患者，模型预测其在下一时间区间 `t_k→t_(k+1)` 是否发生复发，复发定义为 `Normal→Hyper` 的状态转移。

本研究关注的是随访过程中任意一次符合定义的复发转移，而非仅关注首次复发。因此，同一患者在不同随访阶段可多次进入风险集并贡献多个 at-risk intervals。此类重复观测在建模时予以保留，但在交叉验证、阈值选择和测试评估中均按患者分组处理，以控制重复测量带来的相关性。

## 4. Long-format 数据结构

为实现动态预测，本研究将纵向随访数据转换为 person-period 长格式数据结构。每一条记录对应一个时间区间 `(t_k, t_(k+1))`。当患者在区间起点满足 `S_k = Normal` 时，该区间被视为处于复发风险中的有效区间；若在下一时间点满足 `S_(k+1) = Hyper`，则定义该区间结局 `Y_k = 1`，否则定义为 `Y_k = 0`。该数据结构允许单个患者在不同随访区间贡献多个观测样本，同时保证模型仅在存在复发风险的区间上进行学习。基于这一结构，可进一步开展 rolling landmark 预测以及 recurrent-event 对照分析。

## 5. 预测变量

模型纳入三类预测变量。第一类为基线变量，包括人口学特征、甲状腺负荷、RAI 治疗相关信息及免疫学指标。第二类为当前实验室变量，即时间点 `t_k` 可直接获得的指标，包括 `FT3`、`FT4`、`TSH` 及相关血清学指标。第三类为纵向动态特征，用于刻画甲状腺功能随时间变化的模式，包括相对基线变化量、相邻随访时间点差值、既往正常状态持续时间、随访区间指示变量、前一时间点状态，以及既往复发史。

其中，既往复发史定义为当前 landmark 之前是否已经发生过至少一次 `Normal→Hyper` 复发，而非是否在任意既往时间点出现过甲亢状态。除二分类与 rolling landmark 主分析外，在两阶段支持性分析中，还可将对下一窗口生理指标的预测结果作为后续复发判定的补充表示。

## 6. 缺失数据处理与时间安全预处理

缺失数据采用随机森林驱动的迭代插补方法进行处理，可视为 MissForest 风格的实现。预分析中亦曾尝试过更简单的均值插补方案，但在相同的时间安全约束下，其稳健性和后续预测表现均不及当前的 MF-like 方案，因此主分析保留后者。为避免信息泄露，所有数据预处理步骤均遵循时间安全（temporal-safe）策略：首先在患者层面完成训练集与测试集划分；随后仅在训练集上拟合插补模型；最后将训练得到的插补规则应用于测试集。在预测 `t_(k+1)` 时，仅使用不晚于 `t_k` 的信息进行特征构建和数据处理。对于 fixed landmark 任务，输入矩阵按 landmark 深度截断；对于 rolling landmark 与 recurrent-event 相关分析，输入按当前区间起点的可见深度构建。该策略遵循严格 landmarking 原则，保证预测过程尽可能贴近真实临床使用情境。

## 7. 模型构建与验证

为模拟真实临床应用场景，本研究采用时间外推验证。患者按照进入数据库的时间顺序进行排序，在患者级别划分训练集与独立测试集。训练集中采用以患者为分组单位的 `GroupKFold` 交叉验证进行模型训练与超参数调优，以避免同一患者的数据同时出现在训练折与验证折中。超参数通过随机搜索（Randomized Search）优化。分类阈值不在测试集中回调，而根据训练阶段生成的 out-of-fold（OOF）预测概率确定。

固定 landmark 二分类和 rolling landmark 主分析均采用多模型比较策略，候选模型涵盖线性模型、核方法、树模型、平衡集成方法及神经网络模型，最终结果结合验证集表现与临床可解释性综合选择。对于 rolling landmark 这类稀少事件任务，模型比较除关注 `ROC-AUC` 外，亦更重视 `PR-AUC` 对事件识别能力的刻画，并据此确定主报告模型。三分类任务在相同时间安全框架下开展状态分流建模。作为方法学对照，recurrent-event 分析进一步引入 Cox 类模型及随机生存森林，以比较离散时间预测框架与时间到事件框架的一致性。作为支持性分析，两阶段模型先预测下一窗口生理指标，再基于预测结果判断复发风险。

## 8. 模型评估

模型性能在独立的时间外推测试集中进行评估。对于 fixed landmark 二分类和 rolling landmark 任务，主要评价指标包括 `ROC-AUC`、`PR-AUC`、灵敏度、特异性、`F1` 分值及 `Brier` 评分；模型校准通过校准截距、校准斜率及校准曲线进行评估。对关键分类任务，进一步采用按患者分组的自助法估计 `AUC`、`PR-AUC`、`Brier`、灵敏度、特异性及校准参数的 `95%` 置信区间，以描述模型性能的不确定性。为进一步评价临床实用性，还采用决策曲线分析（decision curve analysis, DCA）评估不同风险阈值下的净获益，并通过阈值敏感性分析展示召回率与特异性的权衡关系。rolling landmark 主分析除区间层面评估外，还对多个区间风险进行患者级聚合，用于评价患者在整个后续观察期内发生至少一次复发的总体风险，并在患者级别重复评价其区分度、校准和临床净获益。

对于三分类任务，在报告整体分类性能的同时，采用 one-vs-rest 方式分别评价 `Hyper` 和 `Hypo` 两个重点类别的判别能力、校准能力和临床可用性。对于 recurrent-event 对照分析，则额外报告 `C-index` 以及下一观察窗口风险的区分与校准表现。

## 9. 模型解释性分析

为了提高模型的临床可解释性，本研究将逻辑回归作为重要的可解释性参照模型，重点关注标准化后系数的方向和相对大小，以识别风险升高与风险降低的主要驱动因素。与此同时，作为补充解释路径，还构建树模型和集成模型，并利用 SHAP（Shapley Additive Explanations）方法分析各变量对复发风险预测的贡献方向及重要性。上述解释性分析旨在从不同模型族中识别稳定且具有临床意义的风险信号，而非将单一模型作为唯一解释来源。

## 10. 报告规范、统计分析与应用部署

本研究遵循 TRIPOD-AI 的总体原则进行研究设计和结果报告。所有数据整理、建模和可视化工作均在 Python 环境中完成，主要依赖 `pandas`、`numpy`、`scikit-learn`、`lifelines`、`scikit-survival` 和 `shap` 等工具。本文的方法部分重点描述研究设计、数据组织、建模逻辑与评估框架；统计报告以预测性能、校准与临床效用评估为主，不额外引入与当前实现不一致的显著性检验表述。

为提高模型在临床场景中的可用性，本研究进一步将主分析模型和 fixed landmark 模型封装为基于 Streamlit 的交互式 Web 应用。该应用支持录入患者基线信息、纵向实验室指标及必要的状态历史，并返回个体化风险概率、对应决策阈值以及主要驱动因素摘要，从而为门诊随访中的快速风险沟通和结果解释提供辅助。对于 recurrent-event 和两阶段生理预测等扩展分析，当前主要以 benchmark 展示模块的形式保留，用于补充方法学对照而非一线部署。
